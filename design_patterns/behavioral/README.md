# Behavioral Patterns

Behavioral patterns описват **как обектите комуникират и разпределят отговорностите помежду си**. Фокусът не е върху това как се създават обекти (Creational) или как се структурират (Structural), а върху **потока на управление и данни** между тях.

---

## Patterns

| Pattern | Накратко | Папка |
|---|---|---|
| Chain of Responsibility | Подава заявка по верига от обработчици докато някой я поеме | [chain_of_responsibility](chain_of_responsibility/) |
| Command | Капсулира заявка като обект - поддържа undo, опашки, логване | [command](command/) |
| Interpreter | Дефинира граматика и интерпретира изрази в нея | [interpreter](interpreter/) |
| Iterator | Последователен достъп до елементи без разкриване на вътрешната структура | [iterator](iterator/) |
| Mediator | Централизира комуникацията между обекти - намалява директните зависимости | [mediator](mediator/) |
| Memento | Снимка на вътрешното състояние на обект за последващо възстановяване (undo) | [memento](memento/) |
| Null Object | Замества `null` с обект, който не прави нищо - елиминира null проверките | [null_object](null_object/) |
| Observer | Уведомява множество обекти при промяна на състоянието на Subject | [observer](observer/) |
| State | Поведението на обект зависи от текущото му вътрешно състояние | [state](state/) |
| Strategy | Семейство взаимозаменяеми алгоритми зад общ интерфейс | [strategy](strategy/) |
| Template Method | Базов клас дефинира скелета на алгоритъм, подкласовете попълват стъпките | [template_method](template_method/) |
| Visitor | Добавя операции върху йерархия без да модифицира класовете в нея | [visitor](visitor/) |

## Сравнение на сходни patterns

### Command vs Strategy vs Template Method
| | Command | Strategy | Template Method |
|---|---|---|---|
| Какво варира | КОЯ операция се извършва | КАК се извършва една операция | ЧАСТИ от фиксиран алгоритъм |
| Механизъм | композиция | композиция | наследяване |
| Undo | ✓ (пази се история) | ✗ | ✗ |

### State vs Strategy
И двата pattern-а имат обект, делегиращ поведение към друг обект зад общ интерфейс. Разликата:
- **Strategy** - алгоритъмът се избира **отвън** (от Client) и не се сменя сам
- **State** - преходите между състоянията се управляват **отвътре** (от самия Context или ConcreteState)

### Observer vs Mediator
И двата намаляват директните зависимости между обекти. Разликата:
- **Observer** - Subject знае само за абстрактни Observer-и; комуникацията е едностранна (Subject → Observer)
- **Mediator** - централен обект координира двустранна комуникация между множество обекти; те не знаят един за друг

### Command vs Memento
И двата поддържат undo. Разликата:
- **Command** - знае как да обърне собствената си операция (`undo()`)
- **Memento** - само записва/възстановява snapshot на състояние; не знае какво се е променило

### Visitor vs Iterator
И двата обхождат структура от обекти. Разликата:
- **Iterator** - последователен достъп до елементи, операцията е еднаква за всички
- **Visitor** - различна операция за всеки тип елемент в структурата
